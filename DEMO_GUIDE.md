# 🎯 Гайд для демонстрации проекта

## Подготовка к демо (5 минут до начала)

### 1. Запуск системы
```bash
npm start
# Откройте http://localhost:3000 в браузере
```

### 2. Подготовка Telegram-бота
- Убедитесь, что бот запущен и отвечает на `/start`
- Зарегистрируйте завхоза: `/register Zavhoz Marat`
- Зарегистрируйте 1-2 учителей для демо чата

---

## 🎬 Сценарий демонстрации (по модулям из ТЗ)

### Модуль 1: NLP-Парсер рабочих чатов ✅

**Демо 1.1: Свод по столовой**
1. От имени учителя в Telegram напишите:
   ```
   1А - 25 детей, 2 болеют
   ```
2. Напишите ещё:
   ```
   2B - 70 детей, 7 отсутствуют
   ```
3. На дашборде покажите блок "Питание":
   - Порций: 95
   - Отсутствуют: 9
   - Классов: 2

**Демо 1.2: Менеджмент инцидентов** ⭐ НОВОЕ
1. От имени учителя напишите:
   ```
   В кабинете 12 сломалась парта
   ```
2. Покажите на дашборде:
   - Инцидент появился в списке "Инциденты"
   - Автоматически создалась задача для завхоза в "Задачи"
3. Покажите Telegram завхоза — он получил уведомление:
   ```
   🔧 Новый инцидент
   Кабинет: 12
   Описание: В кабинете 12 сломалась парта
   Приоритет: medium
   ```

---

### Модуль 2: Умное распределение задач ✅

**Демо 2.1: Постановка задач через текст**
1. На дашборде в секции "Задачи" введите:
   ```
   Eldos Seitsamuly, подготовь актовый зал для хакатона.
   Sholpan Shaymuratova, закажи воду и бейджи до пятницы.
   ```
2. Нажмите "Разобрать и поставить задачи"
3. **Покажите защиту от двойных нажатий:** ⭐ НОВОЕ
   - Кнопка блокируется
   - Появляется "⏳ Обработка..."
   - После завершения кнопка разблокируется
4. Покажите результат:
   - Две задачи появились в списке
   - Каждая с правильным исполнителем
   - Есть оценка compliance risk

**Демо 2.2: Уведомления учителям**
1. Покажите Telegram учителя — он получил:
   ```
   Новая задача: подготовь актовый зал для хакатона
   Срок: не указан
   Детали: подготовь актовый зал для хакатона.
   ```

---

### Модуль 3: Динамический менеджмент расписания ✅

**Демо 3.1: Замена учителя**
1. В секции "Замена учителя" выберите:
   - Учитель: `Aidos Sadak`
   - Дата: сегодня (вторник)
2. Нажмите "Показать варианты замены"
3. **Покажите индикатор загрузки:** ⭐ НОВОЕ
   - Кнопка: "⏳ Загрузка..."
   - Защита от повторных нажатий
4. Покажите результат:
   - Система нашла 2 урока физики
   - Предложила свободных учителей
   - Можно выбрать другого из списка
5. Нажмите "Применить замену"
6. **Покажите индикатор:** ⭐ НОВОЕ
   - "⏳ Применение..."
7. Покажите уведомления:
   - Заболевший учитель получил: "Вы успешно заменены..."
   - Заменяющий учитель получил: "Вам назначена замена..."

---

### Модуль 4: Бюрократический RAG ⭐ НОВОЕ

**Демо 4.1: Упрощение приказа**
1. Откройте секцию "Приказы" (новая вкладка в навигации)
2. Вставьте текст приказа (из `src/data/orders.json`):
   ```
   Организация санитарно-эпидемиологических мер, контроль состояния помещений, 
   регулярная фиксация нарушений и оперативное устранение.
   ```
3. Нажмите "Упростить приказ"
4. **Покажите процесс:** ⭐ НОВОЕ
   - Кнопка: "⏳ Обработка..."
   - Защита от двойных нажатий
5. Покажите результат:
   - Gemini переписал приказ в понятный чек-лист
   - Результат в красивом синем блоке
   - Статус: "✅ Приказ упрощён"

---

### Модуль 5: Чат с учителями ✅

**Демо 5.1: Личное сообщение**
1. В секции "Чат" выберите учителя из списка
2. Напишите сообщение: "Не забудьте про родительское собрание"
3. Нажмите "Отправить"
4. **Покажите индикатор:** ⭐ НОВОЕ
   - Кнопка: "⏳"
   - Защита от дублей
5. Учитель получает в Telegram:
   ```
   Сообщение от директора:
   Не забудьте про родительское собрание
   ```

## 💡 Ответы на возможные вопросы жюри

**Q: Почему не все учителя зарегистрированы в боте?**
A: Это демо-окружение. В продакшене будет обязательная регистрация при первом входе.

**Q: Как система обрабатывает конфликты расписания?**
A: Алгоритм в `substitution.js` проверяет занятость учителя в это же время и предлагает только свободных.

**Q: Что если Gemini API недоступен?**
A: Есть fallback-парсер для задач. Для приказов показывается ошибка с предложением повторить.

**Q: Как масштабируется система?**
A: Текущая версия — MVP. Для продакшена нужна замена JSON на PostgreSQL и добавление Redis для кеширования.
