### 1. Автоматический свод в 09:00 (Модуль 1) 🔴
**Приоритет:** ВЫСОКИЙ
**Время:** 30 минут

**Что нужно:**
- Cron-задача или setInterval для проверки времени
- В 09:00 собрать все сообщения о посещаемости за утро
- Отправить свод завстоловой (или директору)

**Реализация:**
```javascript
// В src/server.js
const cron = require('node-cron');

// Каждый день в 09:00
cron.schedule('0 9 * * *', async () => {
  const today = new Date().toISOString().slice(0, 10);
  const attendance = readJson("attendance_records.json", [])
    .filter(r => r.createdAt.startsWith(today));
  
  const summary = buildMealSummary(attendance);
  const message = `📊 Свод питания на ${today}\nПорций: ${summary.totalPortions}\nОтсутствуют: ${summary.totalAbsent}\nКлассов: ${summary.classesCounted}`;
  
  // Отправить завстоловой или директору
  if (directorChatId) {
    await bot.sendMessage(directorChatId, message);
  }
});
```

### 5. Векторная база данных (RAG) 🔴
**Приоритет:** ВЫСОКИЙ (для критерия оценки)
**Время:** 2 часа

**Что нужно:**
- Установить библиотеку для векторного поиска (например, `hnswlib-node`)
- Создать эмбеддинги приказов через Gemini
- Реализовать поиск по релевантности

**Реализация:**
```javascript
// Простая версия без векторной БД (для демо)
async function checkCompliance(taskText, orders) {
  // Поиск ключевых слов в приказах
  const keywords = taskText.toLowerCase().split(' ');
  const relevantOrders = orders.filter(order => 
    keywords.some(kw => order.content.toLowerCase().includes(kw))
  );
  
  if (relevantOrders.length > 0) {
    return {
      risk: "medium",
      reason: `Проверьте соответствие приказам: ${relevantOrders.map(o => o.id).join(', ')}`,
      orderRefs: relevantOrders.map(o => o.id)
    };
  }
  
  return { risk: "low", reason: "Нет конфликтов", orderRefs: [] };
}
```

**Альтернатива для демо:**
- Использовать простой keyword matching
- Сказать: "Полноценный RAG с векторной БД можно добавить через ChromaDB"

### "Почему нет голосового ввода?"

### "Где векторная база данных?"
> Используем Gemini для семантического анализа. Полноценный RAG с ChromaDB — следующий этап.

### "Как масштабируется?"
> Текущая версия — MVP. Для продакшена: PostgreSQL + Redis + очереди + полноценный RAG.
