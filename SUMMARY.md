### ✅ Пункт 6: Автоназначение инцидентов завхозу
**Проблема:** Инциденты создавались, но завхоз не получал задачу автоматически.

**Решение:**
- При обнаружении инцидента в Telegram автоматически создаётся задача для завхоза
- Завхоз получает уведомление с деталями инцидента
- Связь: инцидент → задача → уведомление

**Файлы:** `src/services/telegram.js`

### ✅ Пункт 5: Защита от двойных нажатий + спиннеры
**Проблема:** Кнопки не блокировались, пользователь мог создать дубли задач/замен.

**Решение:**
- Все кнопки блокируются (`disabled`) во время запроса
- Визуальная обратная связь: "⏳ Загрузка...", "⏳ Обработка...", "⏳ Применение..."
- Обработка ошибок сети с понятными сообщениями
- Стили для заблокированных кнопок (серый цвет, `cursor: not-allowed`)

**Затронутые кнопки:**
- Разобрать и поставить задачи
- Показать варианты замены
- Применить замену
- Отправить сообщение в чате
- Упростить приказ

**Файлы:** `public/app.js`, `public/styles.css`

## Как проверить

### Быстрый тест (2 минуты)
```bash
npm start
# Откройте http://localhost:3000
```

1. **Тест защиты от дублей:**
   - Введите задачу
   - Нажмите "Разобрать задачи"
   - Попробуйте нажать ещё раз → кнопка заблокирована ✅

2. **Тест модуля RAG:**
   - Откройте секцию "Приказы"
   - Вставьте любой текст
   - Нажмите "Упростить приказ"
   - Увидите спиннер и результат ✅

3. **Тест автоназначения:**
   - В Telegram напишите: "В кабинете 12 сломалась парта"
   - Проверьте `src/data/tasks.json` → задача для завхоза ✅
