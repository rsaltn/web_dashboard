# ✅ Обновление: Подтверждение исполнителей задач

## Новая функция

### Интерактивный выбор исполнителя
Если система не уверена, кому назначить задачу, появляется меню для выбора правильного исполнителя.

---

## Как работает

### Сценарий 1: Все исполнители найдены ✅
```
Ввод: "Eldos Seitsamuly, подготовь зал"
→ Система находит "Eldos Seitsamuly" в базе
→ Задача создаётся автоматически
→ Уведомление отправляется
```

### Сценарий 2: Исполнитель не найден ❓
```
Ввод: "Елдос, подготовь зал"
→ Система не находит точного совпадения
→ Появляется меню подтверждения
→ Показывается список всех сотрудников
→ Директор выбирает правильного исполнителя
→ Нажимает "Подтвердить и создать"
→ Задача создаётся с выбранным исполнителем
```

## Интерфейс подтверждения

### Карточка задачи:
```
┌─────────────────────────────────────────┐
│ 🔍 Подтвердите исполнителей             │
├─────────────────────────────────────────┤
│ подготовь актовый зал                   │
│ подготовь актовый зал для хакатона      │
│                                         │
│ Исполнитель:                            │
│ [❓ "Елдос" не найден - выберите ▼]    │
│   - Eldos Seitsamuly (ICT)              │
│   - Sholpan Shaymuratova (English)      │
│   - ...                                 │
└─────────────────────────────────────────┘

[✅ Подтвердить и создать] [❌ Отмена]
```

### Цветовая индикация:
- **Зелёная граница** (✅) - исполнитель найден автоматически
- **Оранжевая граница** (❓) - требуется выбор

## Технические детали

### Новый API endpoint
```javascript
POST /api/tasks/confirm
Body: {
  tasks: [
    {
      assignee: "Eldos Seitsamuly",
      title: "подготовь зал",
      details: "подготовь актовый зал",
      due: "",
      complianceRisk: {...}
    }
  ]
}
```

### Изменённый endpoint
```javascript
POST /api/tasks/from-text
// Теперь возвращает задачи БЕЗ сохранения
Response: {
  tasks: [...]
}
```

### Логика на клиенте
```javascript
1. Отправить текст на /api/tasks/from-text
2. Получить распарсенные задачи
3. Проверить каждую задачу:
   - Если assignee есть в списке сотрудников → OK
   - Если нет → показать меню выбора
4. Если все OK → создать автоматически
5. Если есть неопределённые → показать форму подтверждения
6. После выбора → отправить на /api/tasks/confirm
```

## Примеры использования

### Пример 1: Точное имя
```
Ввод: "Eldos Seitsamuly, подготовь актовый зал"
```

### Пример 2: Неточное имя
```

Результат: ❓ Показать меню выбора
Действие: Выбрать "Eldos Seitsamuly (ICT)"
```

```
Результат:

```

### Пример 4: Неизвестное имя
```
Ввод: "Иванов, подготовь зал"
Результат: ❓ "Иванов" не найден - выберите из списка
Действие: Выбрать правильного сотрудника
```

## Преимущества

### 1. Гибкость ✅
- Работает с любыми вариантами написания имён
- Не блокирует создание задач
- Даёт контроль директору

### 2. Прозрачность 🔍
- Видно, какие имена найдены автоматически
- Видно, какие требуют подтверждения
- Можно исправить ошибки до создания

### 3. Удобство 🎯
- Автоматическое создание для точных совпадений
- Интерактивный выбор для неточных
- Отмена в любой момент

### 4. Безопасность 🔒
- Задачи не создаются с неправильными исполнителями
- Всегда можно проверить перед отправкой
- Уведомления идут только правильным людям
